# 평가 (Evaluation)

GAEJO의 변환 품질을 **홀드아웃 케이스**로 측정한 방법과 기준 결과.

## 방법

- **데이터**: `examples/eval_cases.json` — 구어체 한국어 연구 발표 발화 10개(합성, few-shot과 미중복).
  bullet/title/slide 단위와 규칙 커버리지(수치·근사 뉘앙스·인과·대조·영어 용어·제목 라벨화)를 분산.
- **변환**: `gaejo.transform.transform` (현 기본 모델 `claude-opus-4-8`, augmented zero-shot + 합성 few-shot).
- **평가 3축**:
  - **스타일(객관)**: `gaejo.score` — 개조식 종결 비율·완전문장 잔존. Kiwi 규칙, BLEU/ROUGE 비의존.
  - **의미 보존(객관)**: `gaejo.content_retention(original, output)` — 수치·전문용어·헤지어 보존을
    형태소 규칙으로 측정(키 불필요). `gaejo.objective`가 `retention` 키로 포함.
  - **스타일·의미보존·자연스러움(주관)**: LLM 3축 판정 앙상블(케이스당 독립 판정 3인 평균).
- **재현**: `ANTHROPIC_API_KEY=... python examples/evaluate.py --judge`

### 객관 의미 보존 메트릭 (`content_retention`)

LLM 판정은 비싸고 모델 의존적이라, 측정된 주된 손실(수치·전문용어·뉘앙스 누락)을 **키 없이** 잡는
객관 게이트를 둔다. 원문→출력 보존 여부를 세 갈래로 본다:

| 갈래 | 검사 | 예 |
|---|---|---|
| **수치** | 아라비아 숫자 + 한국어 수사('두'→2). 값이 출력에 남았나 | "4.7%p" 누락 → 0.0 |
| **전문용어** | 라틴 토큰(2자+, 'LoRA'·'attention'). 강제 번역 금지 규칙 검증 | "attention" 탈락 → 0.0 |
| **헤지어** | 근사/역접/강조/가능성 4범주. 범주 단위 보존('정도'→'약'은 둘 다 근사 → 보존) | "반드시" 탈락 → 강조 범주 손실 |

이 메트릭은 평가에서 LLM 판정자가 지적한 손실(예: "반드시"·"굉장히" 강조 탈락)을 그대로 객관적으로
재현한다 — 의미 보존 약점을 키 없이 회귀 추적할 수 있다.

## 기준 결과

| 축 | 점수 |
|---|---|
| 스타일 변환 정확도 (LLM 앙상블) | **0.97** |
| 의미 보존 (LLM 앙상블) | **0.92** |
| 자연스러움 (LLM 앙상블) | **0.95** |
| 개조식 종결 비율 (객관/Kiwi) | **100%** |
| 완전문장 잔존 (안티패턴) | **0건** |

예시 변환:

```
"...정확도를 4.7%p 정도 끌어올렸습니다"        → "제안 모델, 기존 baseline 대비 accuracy 약 4.7%p 향상"
"...방법론에 대해서 설명드리겠습니다"           → "제안 방법론"                       (제목: 라벨형)
"노이즈가 많아지니까 ... 성능이 떨어지는..."     → "데이터 노이즈 증가 → 모델 일반화 성능 저하"
```

## 남은 약점·관찰

- **영/한 표기 혼용 일관성**: 'accuracy'·'benchmark'를 원어로 둘지 한자어로 옮길지 케이스마다 흔들림.
  이는 "영어 전문용어 원어 유지가 기본값"이라는 코퍼스 규칙(→ [methodology.md](methodology.md))의 의도된
  trade-off에 가깝다. 일관성을 강제하려면 도메인 용어 사전을 별도로 둬야 한다.
- **강조 부사 약화**(measured): "굉장히/반드시" 탈락은 이제 `content_retention`의 헤지 범주 손실로
  객관 포착된다 — 프롬프트의 헤지 보존 규칙을 강화하면 회귀로 검증 가능.
- **제목의 발화행위 생략**: "설명드리겠습니다" 같은 메타 발화가 라벨형 제목에서 탈락 → 의미보존 점수에
  소폭 반영되나, 한국 발표 관행상 정상(제목=라벨형, 주장은 본문).
- **강조 부사 약화**: "굉장히/반드시"가 "대량/필요함"으로 압축되며 강도가 다소 줄 수 있음.
- LLM 판정은 모델 의존·편향(self-preference 등)이 있어 앙상블·다모델로 완화 권장.

## HIL(Human-in-the-loop)과 judge 교정

LLM 판정은 값싸지만 모델 의존·편향이 있어, 결국 **사람 정답으로 검증**해야 한다. GAEJO는 이를
신뢰도 캐스케이드로 엮는다:

```
transform → ① 객관 게이트(score+retention) → ② LLM judge(3축) → ③ HIL(사람 승인/수정)
                                                                     └→ gold.jsonl
```

`gaejo review`로 사람이 후보를 승인/수정/기각하면 `(원문, 후보, 객관, judge, 사람최종)`이 gold로 쌓인다.
`gaejo.hil.judge_agreement(gold)`는 **judge 점수 ↔ 사람 결정**의 일치도를 낸다:

- `pearson_r` — judge 평균점수와 사람 품질라벨(accept 1 / edit .5 / reject 0)의 상관
- `accept_accuracy`, `cohen_kappa` — judge가 임계값(기본 0.85) 이상이면 '수락'으로 예측했을 때 사람 실제 수락과의 일치

> **왜 필요한가 (실측):** 3케이스 데모에서 judge 평균은 셋 다 0.85 이상이라 단독 게이트면 **전부 자동 수락**했겠지만,
> 사람은 '반드시'(강조) 손실 케이스를 수정했다. 같은 케이스를 **객관 retention은 0.0으로 정확히 표시**했다.
> 결과적으로 judge 단독 accept-게이트의 κ는 0(관대함)으로 나와, 객관 게이트 + HIL 보강의 필요가 정량으로 드러난다.

## 비고

DSBA 코퍼스에서 추출한 변환쌍으로 측정한 초기 프로토타입 대비(스타일 0.95·의미보존 0.89·자연 0.92),
저작권 회피를 위한 **합성 few-shot 교체 후에도 품질이 유지·향상**됨을 확인했다(의미보존 개선은 프롬프트의
근사·역접 뉘앙스 보존 규칙 보강에 기인). 표본 10건의 소규모 평가이므로 절대 수치는 참고치로 본다.
