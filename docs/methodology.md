# 방법론 — GAEJO 규칙은 어떻게 도출됐나

GAEJO의 변환 규칙은 두 단계로 만들어졌다: **(1) 문헌 리서치**로 가설을 세우고,
**(2) 한국 연구 발표 슬라이드 코퍼스 분석**으로 실측 보정했다. 이 문서는 그 근거와 통계를 기술한다.
원본 슬라이드 텍스트는 저작권상 포함하지 않으며, 집계 통계와 규칙만 제시한다.

## 1. 코퍼스

- 출처: 공개된 한국 대학원 연구실의 논문 리뷰 세미나 발표 영상(공학·NLP·비전 등 다도메인).
- 수집: 발표 영상에서 장면 전환 키프레임 추출 → 슬라이드 텍스트 분석.
- 규모: 유효 슬라이드 약 55장, 한국어 개조식 줄 약 267개, 슬라이드 제목 약 77개.
- 검증: 종결 양식 판별을 형태소 분석기(Kiwi)로 자동화하고, 사람/모델 라벨과 약 **96%** 일치 확인.

> 본 저장소에는 코퍼스 원본·추출 텍스트를 포함하지 않는다. few-shot 예시는 모두 독창적 합성 데이터다.

## 2. 실측 종결 분포 (한국어 줄 기준)

| 종결 양식 | 비율 | 비고 |
|---|---|---|
| 순수 명사 종결 | ≈ 75% | **디폴트** |
| `-ㅁ/음` (명사형) | ≈ 24% | 완결 서술절·저자판단 |
| `-기` (명사형) | **0%** | 종결로는 미사용 (전부 "-기 위해" 연결) |
| 완전문장 (~다/~습니다) | ≈ 2% | 예외(RQ·인사) |

## 3. 형태소 처리 — Kiwi EF/ETN 불일치

음슴체 `-ㅁ`은 어간 품사·문장부호에 따라 Kiwi가 **EF(종결어미)** 또는 **ETN(명사형 전성어미)**으로
엇갈리게 태깅한다.

| 표면형 | 토큰화 | 종결 태그 |
|---|---|---|
| 개선함 | 개선/NNG 하/XSV ᆷ | **EF** |
| 강건함 | 강건/XR 하/XSA ᆷ | **ETN** |
| 수집함. | 수집/NNG 하/XSV ᆷ . | **ETN** |
| 검증하기 | 하/XSV 기 | **ETN** |

따라서 "ETN 태그로 개조식을 검출"하는 단순 규칙은 음슴체를 놓친다. GAEJO는 **태그 + 어미 형태(form)**를
함께 보는 결정 트리를 쓴다.

### 종결 결정 트리 (`gaejo.detector`)

```
줄 끝(후행 부호 제거) 마지막 형태소:
  ├─ 한글 없음              → 영문 (변환 대상 아님: 헤더/용어)
  ├─ ETN(전성어미)
  │     ├─ form '기'        → 기   (개조식)
  │     └─ form ㅁ/음       → ㅁ음 (개조식)
  ├─ EF(종결어미)
  │     ├─ form ㅁ/음       → ㅁ음 (음슴체가 EF로 태깅된 경우 구제)
  │     ├─ form '기'        → 기
  │     └─ 그 외(다/습니다/까/요…) → 완전문장 (비개조식)
  ├─ 조사(JK*/JX/JC)        → 기타 (미완성)
  ├─ 체언/외국어/숫자/한자   → 명사 (개조식, 디폴트)
  └─ 그 외(용언 어간 등)     → 기타
```

## 4. 리서치 가설 → 실데이터 판정

| 리서치 가설 | 코퍼스 판정 | 메모 |
|---|---|---|
| `-음`=사실, `-기`=방법/미정 | **부분 반박** | `-기` 종결 0건. 방법·절차도 명사 종결로 처리 |
| 종결 핵심은 명사형 어미(`-ㅁ/음`) | **수정** | 핵심은 맞으나 디폴트는 **순수 명사 종결** |
| 조사 생략 | **무조건→조건부** | 목적격 보존:생략 ≈ 1:1, 명사구화 가능 시만 생략 |
| 군더더기·접속어 제거 | **엄격→부분압축** | 대조/전환 논리는 의도적으로 유지 |
| 제목=주장형 헤드라인(Assertion-Evidence) | **반박** | 주장형 제목 0건. 라벨형 지배. 주장은 본문으로 |
| 불릿당 ≤10단어 | **부분 반박** | 한·영 혼용 → 어절 중앙값 ≈7, 상한 ≈25토큰 기준 |
| 슬라이드당 3~5불릿 | **확인** | 평균 ≈4.7줄(헤더 포함) |
| ETN 태그로 개조식 검출 | **반박** | EF/ETN 불일치 → 태그+어미형태 병용 |

## 5. 신규 규칙 (리서치엔 없던, 실데이터에서 발견)

1. **명사 vs -ㅁ음 변별 = 통사적 완결성** (의미 아님): 명사구만 → 명사 종결, 완결 서술절 → `-ㅁ음`.
2. **주격조사 흡수 패턴**: `명사 + 서술명사(존재/발생/저하/향상/필요)`.
3. **기호로 군더더기 외주화**: 화살표(→/⇒/∝)=인과·비례·귀결, 콜론(:)=라벨-정의.
4. **한 줄 2단 압축**: 위계를 더 파지 않고 "X: 설명", "A → B"로 처리.
5. **3단계 위계 정상 사용**(영어권 2단계 선호와 다름), 말단일수록 길어짐.
6. **영어 전문용어 원어 유지가 기본값**, 명사화 우선(능동/수동 구분보다 우선).

## 6. 평가 방식

TST 평가는 다차원이며 BLEU/ROUGE는 인간 판단과 상관이 낮다. GAEJO는 3축으로 평가한다.

- **스타일(객관)**: `gaejo.score` — 개조식 종결 비율·어절 수·완전문장 안티패턴(Kiwi 규칙).
- **의미보존·자연스러움(주관)**: LLM 판정(가능하면 다모델 앙상블, 편향 보정).
