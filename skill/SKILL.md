# GAEJO — 개조식 변환 하네스

한국 학술/기술 발표 슬라이드의 **개조식** 문체로 텍스트를 다듬는다. **LLM(너, 에이전트)이 변환하고**,
`gaejo` CLI로 **결정론적으로 검증**한다(형태소 분석 기반, Kiwi). 변환 품질의 책임은 너에게 있고,
GAEJO는 규칙과 채점·검증을 제공한다.

## 전제

`gaejo` CLI가 설치돼 있어야 한다(`pip install -e ".[kiwi]"` 또는 `pip install gaejo[kiwi]`).
설치 확인: `gaejo --version`. 없으면 사용자에게 설치를 안내한다.

## 워크플로 (이 순서를 지켜라)

1. **변환**: 아래 [규칙]에 따라 구어체/완전문장 → 개조식으로 직접 바꾼다(줄·위계 보존).
2. **검증**: 각 줄(또는 블록)을 `gaejo check`로 확인한다.
   ```bash
   gaejo check "<원문>" "<변환본>"
   ```
   → `{"ok": bool, "issues": [...], "gaejo_ending_ratio":, "content_retention":, ...}`
3. **수정**: `ok:false`면 `issues`를 그대로 반영해 다시 변환하고 2를 반복한다.
   - blocks(여러 줄)는 `gaejo score "<여러 줄>"`로 한 번에 채점 가능.
   - 한 줄 종결만 빠르게 볼 땐 `gaejo detect "<줄>"`.

**원문 한 줄도 빠뜨리지 말고 검증한다. issues를 임의로 무시하지 말 것.**

## 규칙 (개조식 핵심 — 코퍼스 분석으로 보정)

**종결 양식** (실데이터 분포: 명사 ≈75% · -ㅁ/음 ≈24% · -기 0% · 완전문장 ≈2%)
- 기본값 = **순수 명사 종결**. 명사구만 제시할 땐 체언으로 끝낸다. 예) "성능 향상", "한계 존재".
- **-ㅁ/음 종결**은 (a) 주어·부사어를 거느린 완결 서술절 또는 (b) 저자의 수행·판단·당위일 때만.
  예) "제안함", "필요함", "보여줌", "달라짐".
- **'-기'를 종결로 쓰지 않는다**(코퍼스 0건). '-기'는 "측정하기 위해" 같은 연결로만.
- **완전문장(~다/~이다/~한다/~습니다/~는가?/~하자) 금지.** 단 다음은 그대로 두라(개조식으로 바꾸지 말 것):
  연구질문(RQ)·인사말·**슬라이드 헤드라인 의문형**("…무엇인가?")·**인용 예시**(따옴표 안 자연어 명령/대사).
  `gaejo check`가 이들을 "완전문장"으로 잡아도 의도된 예외이므로 무시 가능.
- 같은 부모 아래 형제 항목은 종결 양식을 일치시킨다.

**압축**
- 용언 활용 어미를 잘라 명사/명사형 종결로(1차 압축).
- 조사 생략은 **조건부**: 명사구화 가능 시 생략("최대 batch 사용"), 절차·인과 서술 시 보존("PCA를 도입").
- 주격조사는 '명사 + 서술명사(존재/발생/저하/향상/필요)' 결합으로 생략("성능 차이 발생").
- 군더더기를 줄이되 **일괄 삭제 금지** — 대조·전환·인과 논리(그러나/다만/반면)는 필요 시 남긴다.
- 인과·비례·귀결은 화살표(→, ⇒, ∝), 라벨-정의는 콜론(:)으로 외주화("파라미터 ∝ 성능").

**어휘**
- 영어 전문용어는 **원어 유지**(LoRA, transformer, ROUGE…). 정착 한자어 있을 때만 번역. 강제 번역 금지.
- '영어 명사구 + 한국어 조사/술어' 골격(rank가, block을). 명사화 우선(능동/수동 구분보다).

**제목**
- 슬라이드 제목 = **라벨형 명사구**(종결 어미 금지). 예) "제안 방법", "실험 결과".
- 주장(평서문)은 제목이 아니라 본문 글머리표로.

**보존(중요 — 측정된 약점)**
- 원문의 수치(4.7%p)·고유명사·영어용어는 반드시 보존.
- **근사(약/정도)·역접(다만/오히려)·강조(반드시/굉장히) 뉘앙스를 함부로 떨어뜨리지 마라.** `gaejo check`가 이 손실을 잡는다.

## 예시

```
"요즘 모델들이 성능은 좋은데 파라미터가 커서 서비스에 올리기 어렵습니다"
  → "성능은 우수하나 파라미터 과대 → 서비스 적용 곤란"

"먼저 가상에서 충분히 검토한 뒤 실제 제작 리스크를 줄이는 데 목적이 있습니다"
  → "가상 환경 충분 검토 → 실제 제작 리스크 감소가 목적"

"저희가 제안하는 방법을 설명드리겠습니다"   → "제안 방법"   (제목: 라벨형)

"ROUGE를 거의 비슷하게 유지했는데 학습 비용은 굉장히 줄었습니다"
  → "ROUGE 거의 동등 유지 / 다만 학습 비용 대폭 감소"   (거의·다만 보존, ROUGE 원어)
```

## 자기검증 예 (이 루프를 돌려라)

```bash
$ gaejo check "모델 경량화가 반드시 필요합니다" "모델 경량화 필요함"
{"ok": false, "issues": ["뉘앙스 누락 범주: ['강조'] — 해당 뉘앙스를 살려라"], ...}
# → '반드시' 복원: "모델 경량화 반드시 필요함" 으로 다시 변환 후 재검증
$ gaejo check "모델 경량화가 반드시 필요합니다" "모델 경량화 반드시 필요함"
{"ok": true, "issues": [], ...}
```

## 주의

- 이건 자동 변환이 아니라 **너의 변환 + 결정론적 검증**이다. 의미를 바꾸거나 비문을 만들지 마라.
- 자료 전체를 다룰 땐 원본을 덮지 말고 before→after로 제안하라(사람 검수 필요).
- 더 자세한 규칙·few-shot이 필요하면 `gaejo prompt "<문장>"`로 패키지 프롬프트 전체를 볼 수 있다.
